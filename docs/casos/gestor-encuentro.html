Volver a migraciones GobTur Platform · Mayo 2026

Gestor de Encuentros — GobTur Platform

Programa de migración del legacy del Gestor de Encuentros (y visión unificada con RIS) hacia un ecosistema de microservicios con responsabilidades claras, integraciones centralizadas y despliegues por dominio.

7 fases ~32% avance Oracle GOB_TUR GCBA

Resumen ejecutivo

El Gestor de Encuentros y RIS comparten dominio de negocio (agenda, turnos, integraciones con el ecosistema de salud) pero mantenían backends duplicados y múltiples puntos de salida hacia sistemas externos (Master File, Sigehos, HIS…).

GobTur Platform transforma el monolito del Gestor en microservicios especializados. Cuando el tramo esté probado en producción, se incorporará la lógica de RIS back en los mismos servicios: un solo ecosistema para el front del Gestor y el front de RIS.

7 Microservicios + librería común
1 Único egress a terceros (8082)
~32% Avance global del programa

De monolito a microservicios

Hoy (legacy)

  • Gestor front → Monolito Gestor de Encuentros
  • RIS front → RIS back
  • Ambos conectan a MF, Sigehos, HIS por separado
  • Reglas duplicadas y despliegues de alto riesgo

Objetivo (GobTur Platform)

  • Gestor front + RIS front → Gateway → MS
  • external-services (8082) único punto a terceros
  • Una regla, un lugar — despliegue por dominio
  • Escalado selectivo (agenda, reportes…)

Etapa 2

Microservicios Standalone (Jun 2026)

Una vez consolidado el ecosistema GobTur, se ejecutó el desacople de cada servicio del reactor gobtur-platform y la dependencia gobtur-common: cada carpeta bajo gdea-back/ pasó a ser un proyecto Maven independiente, listo para su propio repositorio.

Procedimiento por servicio

  1. Copiar fuentes Java de gobtur-common al servicio
  2. Copiar recursos (application-gobtur-*.properties, rutas BFF)
  3. Reescribir pom.xml: parent spring-boot-starter-parent 3.5.6, Java 21
  4. Compilar de forma aislada: mvn -f {servicio}/pom.xml clean package
  5. Eliminar reactor y módulo common tras migrar los 9 servicios

Visión RIS (programa futuro)

Decisión mayo 2026: cerrar primero las Fases 0–7 del Gestor. La integración de RIS back al mismo ecosistema GobTur es el programa siguiente — no bloquea la Fase 2 ni 3 actual.

Fases 0–7 Gestor → Inventario RIS back → Migrar módulo a MS dueño → RIS front → MS → Retirar RIS back

Valor de negocio

Menos duplicación

Fin de arreglar la misma regla en Gestor y RIS.

Despliegues seguros

Actualizar solo agenda o solo externos, sin redeploy total.

Seguridad

Credenciales MF/Sigehos/HIS concentradas en un solo servicio.

Evolución incremental

Olas de deploy, flags y hub de migración para steering.
