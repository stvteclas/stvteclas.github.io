Gestor de Encuentros — Microservicios Standalone

Desacople de 9 microservicios del reactor gobtur-platform y la dependencia compartida gobtur-common, siguiendo el patrón de laboratorio-service: parent directo spring-boot-starter-parent y código común embebido en cada servicio.

Spring Boot 3.5.6 Java 21 9 microservicios Maven standalone Resilience4j

Visión general

La plataforma gdea-back evolucionó de un monorepo Maven multi-módulo hacia 9 proyectos Maven independientes, cada uno listo para moverse a su propio repositorio. El reactor padre (gdea-back/pom.xml) y el módulo gobtur-common fueron eliminados tras completar la migración.

9 Microservicios standalone
0 Dependencias de reactor padre
100% Compilación aislada por servicio

Arquitectura resultante

Antes

Monorepo Maven
  • gobtur-platform (reactor)
  • gobtur-common (shared)
  • 9 MS acoplados al parent

Después

9 proyectos independientes
  • spring-boot-starter-parent 3.5.6
  • Java 21 por servicio
  • Common embebido localmente
  • Listo para repo dedicado

Procedimiento por servicio

Paso 1

Copiar fuentes de gobtur-common

  • ar/gob/gcba/his/gobtur/common/** → misma ruta en el servicio
  • Subpaquetes comun, exception, model sin pisar archivos locales
Paso 2

Copiar recursos compartidos

  • application-gobtur-*.properties
  • gobtur-bff-routes.yaml
  • Merge de AutoConfiguration.imports
Paso 3

Reescribir pom.xml

<parent>
  spring-boot-starter-parent 3.5.6
</parent>
<groupId>ar.gob.gcba.his</groupId>
<java.version>21</java.version>
// Sin dependencia gobtur-common
Paso 4

Compilar de forma aislada

mvn -f gdea-back/{servicio}/pom.xml clean package
# Requiere JDK 21 (JAVA_HOME)
Paso 5

Eliminar reactor y common

Tras migrar los 9 servicios, se eliminaron gdea-back/pom.xml y gobtur-common. Cada servicio compila de forma totalmente independiente.

Casos especiales

Dueño exclusivo del circuito de licencias tras el aislamiento:

  • REST: /licencia/**, /turnocancelacionlote/**
  • Motor agenda delegado vía HTTP a agenda-service (:8083)
  • BFF enruta vía LicenciaMsClient
gobtur.remote.agenda-base-url=

Auto-configuración embebida

Al embeber el código de gobtur-common, se activan automáticamente:

GobturHttpClientAutoConfiguration GobturExternalBridgeAutoConfiguration GobturBffAutoConfiguration GobturWebAutoConfiguration

Prueba de humo (licencia-service)

  2. Misma BD Oracle y gobtur.internal.api-key compartida
  3. Health: GET .../actuator/health
  4. Con JWT válido: GET .../licencia/findHoy
